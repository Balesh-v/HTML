HTML File Paths

File paths are used to link to other documents.

Absolute File Paths

An absolute file path contains the full URL to another document, including the protocol (http:// or https://).
Example:

<a href="https://www.example.com/page1.html">This is a link</a>

Relative File Paths

A relative file path contains only the file name or folder name(s) and the file name.
The link is relative to the current page.
Example:

<a href="page1.html">This is a link</a>

<a href="folder/page1.html">This is a link</a>

<a href="../page1.html">This is a link</a>

Root Relative File Paths

A root relative file path starts with a forward slash (/) and contains the full path from the root of the website.
Example:

<a href="/folder/page1.html">This is a link</a>

File Path Tips

  • Use absolute paths for links to other websites.
  • Use relative paths for links within your own website.
  • Use root relative paths if you want to ensure that the link works regardless of the current page's location within the site structure.

Note

File paths are case-sensitive on most web servers.
Always use the correct case when specifying file and folder names in your paths.

Example

<a href="https://www.example.com/page1.html">This is a link</a>

<a href="page1.html">This is a link</a>

<a href="folder/page1.html">This is a link</a>

<a href="../page1.html">This is a link</a>

<a href="/folder/page1.html">This is a link</a>

Conclusion

Understanding file paths is essential for creating effective links in your HTML documents. By using the appropriate type of file path, you can ensure that your links work correctly and provide a better user experience.
